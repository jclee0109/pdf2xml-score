# PDF 악보 → 정확한 MusicXML 변환기 MVP PRD

## 1. 문서 목적
본 문서는 **PDF/이미지 형태의 악보를 기존 악보 프로그램에서 바로 활용 가능한 clean MusicXML로 변환**하기 위한 MVP의 개발 범위를 정의한다.  
초기 목표는 **완전한 악보 편집 툴을 만드는 것**이 아니라, **AI 변환 결과를 사람이 빠르게 검수해서 실제로 쓸 수 있는 파일을 만드는 레이어**를 만드는 것이다.

---

## 2. 배경 및 문제 정의

### 실제 사용자 시나리오
기획사 작곡가·편곡가는 다음과 같은 워크플로우를 반복한다.

1. 인터넷에서 악보 PDF를 구한다 (보컬 악보, 피아노+보컬, 리드시트 등)
2. MuseScore / Dorico 등 악보 프로그램에서 활용하려 한다
3. PDF → MusicXML 자동 변환을 시도하지만 결과가 너무 깨져 있다
4. 결국 악보를 보면서 처음부터 손으로 다 입력한다

**자동 변환 결과가 너무 부정확해서, 고치는 시간이 처음부터 입력하는 시간과 같다.**

### 현재 문제
- PDF는 보기에는 편하지만 악보 프로그램에서 편집·활용이 불가능하다
- 기존 PDF → MusicXML 변환 결과에 오류가 너무 많다
  - 코드 심볼이 가사로 들어가거나 아예 사라짐
  - 가사와 텍스트가 뒤섞임
  - 조표 / 키 감지가 틀림
  - 마디 구조가 어긋남
  - 반복 기호 깨짐
- 어디가 틀렸는지 파악하기도 어렵고, 결국 수동으로 다시 작업한다

### 핵심 인사이트

> **PDF를 완벽하게 자동 인식하는 것**보다,  
> **AI가 80% 변환하고, 사람이 나머지 오류를 빠르게 검수해서 실제로 쓸 수 있는 파일을 만드는 것**이 더 현실적이고 가치 있다.

## 3. 경쟁사 분석

### 기존 도구의 한계

| 도구 | 형태 | 가격 | 코드 심볼 | 검수 UI | 주요 문제 |
|------|------|------|----------|--------|---------|
| SmartScore | 데스크탑 | $300+ | **없음** | 제한적 | 수동 수정 50%+, 학습 곡선 |
| PhotoScore | 데스크탑 | $200+ | **없음** | 제한적 | 오류율 높음, 자주 오배치 |
| PlayScore 2 | 모바일 | 프리미엄 | 기타코드만 | 없음 | key 변경 감지 실패 |
| Audiveris | 오픈소스 | 무료 | **없음** | 존재하나 복잡 | 개발자 전용, 핸드라이팅 불가 |
| MuseScore import | 내장 | 무료 | 거의 동작 안 함 | 없음 | 실험적 기능 수준 |
| Melogen AI / PDFtoMusicXML | 온라인 | 무료 | 없음 | **전무** | 변환만 하고 끝 |

### 공통된 실패 패턴
기존 도구들은 두 가지를 동시에 포기했다:
1. **코드 심볼 처리** — 어떤 상용 도구도 제대로 인식하지 못함
2. **검수 경험** — 변환 결과를 빠르게 검수·수정하는 UX가 없거나 매우 불편함

실사용자 후기: *"프루프리딩에 처음부터 입력하는 시간만큼 걸린다"*, *"SmartScore 써도 50% 이상 수동 수정"*

### 시장 공백 요약
> **정확한 변환 결과 + 빠른 검수 UI** = 현재 어떤 도구도 제공하지 않는 조합

## 5. 타겟 사용자

### 1차 타겟
- 기획사 작곡가 / 편곡가
- 세션 연주자
- 보컬 (키 전조 목적)
- 실용음악 전공생 / 강사
- 싱어송라이터
- 교회 반주자 / CCM 편곡자

### 대표 사용 시나리오
- 구해온 PDF 악보를 MuseScore에서 바로 활용하고 싶다
- 전조 / 코드 수정 / 가사 수정을 악보 프로그램에서 하고 싶다
- PDF를 Dorico / Sibelius에서 이어서 편집하고 싶다
- 악보 변환 결과물이 너무 깨져서 고치는 시간이 아깝다

## 6. MVP 범위

### 포함 범위
- 리드시트 (단선율 + 코드 심볼)
- 보컬 악보 (멜로디 + 코드 심볼 + 가사)
- 피아노+보컬 2단 편성 (검토)
- 찬송가 / CCM류 단순 편성
- 비교적 깨끗한 PDF 또는 이미지 입력
- MusicXML export

### MVP 제외 (현재 단계)
- 오케스트라 풀스코어 / 복잡한 피아노 솔로 / 3성부 이상 복잡 편성 → **Phase 2에서 지원**
- 특수 현대악보 기호
- 완전한 notation editor
- DAW 수준 재생 / 편집 기능

### MVP 지원 기준
> AI가 80% 이상 구조를 추출할 수 있고, 나머지 오류를 사람이 빠르게 검수 완료할 수 있는 악보

### 확장 설계 원칙
복잡도 확장은 제품의 필수 방향이다. MVP 설계 단계부터 아래를 고려한다:
- 악보 복잡도(성부 수, 악기 수)에 따라 검수 항목이 동적으로 달라지는 구조
- 단순 악보 → 복잡 악보로 갈수록 검수 시간이 늘어나는 것을 전제한 UX 설계
- 내부 데이터 모델이 풀스코어의 다성부 구조를 처리할 수 있도록 확장 가능하게 설계

## 7. 핵심 문제

### 문제 1. 기존 변환 도구가 코드 심볼을 처리하지 못한다
- 어떤 상용 OMR 도구도 코드 심볼을 정확히 인식하지 못함
- 코드 심볼이 가사로 혼입되거나, 아예 누락됨
- 코드가 틀리면 전조 / 편곡 / 반주 생성 모두 무의미해짐

### 문제 2. 변환 결과에 오류가 많고 어디가 틀렸는지 알기 어렵다
- 조표 / 키 감지 오류
- 가사 ↔ 텍스트 혼재
- 마디 구조 어긋남
- 반복 기호 깨짐
- 오류가 분산되어 있어 검토 자체가 시간을 많이 잡아먹음

### 문제 3. 수동 수정 비용이 너무 크다
- 기존 도구는 변환만 하고 검수 UX를 제공하지 않음
- 결국 변환 결과를 보면서 처음부터 손으로 다시 입력하게 됨

---

## 8. MVP 가치 제안

### 기존 방식
PDF 업로드  
→ 범용 OMR 변환 (코드 심볼 누락, 오류 다수)  
→ MuseScore에서 깨진 결과 확인  
→ 처음부터 손으로 다시 입력  

### 우리 방식
PDF 업로드  
→ AI 1차 변환  
→ 오류 항목만 빠르게 검수 (코드 심볼 / 조표 / 가사 / 마디)  
→ clean MusicXML export  
→ MuseScore / Dorico에서 바로 활용  

### 핵심 가치
- **"변환만 해주는 도구"가 아닌 "쓸 수 있는 파일을 만들어주는 도구"**
- 완벽 자동화보다 **수정 시간 단축**
- 범용 편집기 대체가 아니라 **기존 툴 앞단의 품질 개선**
- 코드 심볼까지 처리하는 유일한 상용 레이어

## 9. 핵심 기능

### 9.1 입력
- PDF 업로드
- 이미지 업로드 (PNG / JPG)
- 단일 파일 기준 처리

### 9.2 AI 1차 변환
- 입력 파일을 구조화된 score 데이터로 변환
- 내부 intermediate representation 생성
- 오류 가능성이 높은 항목에 자동으로 플래그 부여

### 9.3 검수 UI ★ (핵심 차별점)

#### 설계 방향 (미결 — UX 상세 설계 필요)
- 인식된 악보를 화면에 렌더링
- 오류 의심 항목을 시각적으로 하이라이트
- 원본 PDF를 참조용으로 나란히 제공
- 클릭해서 인라인 수정

#### 검수 항목 우선순위

**A. 코드 심볼 검수 (1순위)**
- 인식된 코드 심볼 목록 표시
- chord / lyric / text 후보 구분
- 오인식 수정
- 코드를 구조화된 형태로 저장 (전조 / 편곡 가능하도록)

**B. 조표 / key 검수 (2순위)**
- 감지된 key 표시
- 사용자가 key 수정 가능

**C. 가사 / 텍스트 분리 (3순위)**
- lyric / chord / section mark 분류 확인
- 잘못 분류된 텍스트 수정

**D. 마디 구조 검수 (4순위)**
- 마디 수 표시 및 누락 / 병합 후보 알림
- 반복 기호 검수

### 9.4 Export
- MusicXML 다운로드
- MuseScore / Dorico / Sibelius import 전제
- (선택) 전조본 preview

## 11. 기능 명세 초안

### 11.1 Upload Module
#### 목적
입력 파일을 수신하고 변환 파이프라인에 전달한다.

#### 요구사항
- PDF / JPG / PNG 지원
- 파일 크기 제한 설정
- 업로드 성공 / 실패 처리

---

### 11.2 Parse / Convert Module
#### 목적
PDF / 이미지에서 score 구조를 추출하여 내부 데이터로 변환한다.

#### 출력
- measures, notes, lyrics, chord symbols
- key signature, time signature
- repeats / section marks

#### 비고
- 초기에는 외부 OMR 엔진 + AI 레이어 조합 검토
- "완벽한 변환"보다 검수가 가능한 구조 데이터 생성이 목적
- 오류 가능성이 높은 항목에 플래그 자동 부여

---

### 11.3 Review Module
#### 목적
사람이 최소한의 시간으로 오류를 찾고 수정할 수 있도록 한다.

#### UX 원칙
- 전체 score 편집기가 아닌 "오류 중심 검수 UI"
- 수정 결과가 export에 직접 반영
- 목표: A4 1페이지 기준 검수 완료 5분 이내

---

### 11.4 Export Module
#### 목적
수정된 결과를 외부 악보 프로그램에서 바로 사용할 수 있도록 내보낸다.

#### 출력
- MusicXML
- (선택) Transposed PDF preview

#### 성공 기준
- MuseScore / Dorico import 후 추가 수정 없이 바로 활용 가능할 것
- chord / lyric / key 정보가 정상 반영될 것

## 12. 데이터 모델 초안

### ScoreDocument
- id, original_file_path, created_at, status

### ScoreMeta
- title, composer, key_signature, time_signature, tempo

### Measure
- index, repeat_start, repeat_end, section_label

### Note
- pitch, duration, measure_index, voice, lyric_ref

### Lyric
- text, note_ref, position

### ChordSymbol
- text, normalized_value, measure_index, beat_position

### ReviewIssue
- type (`chord`, `key`, `lyric`, `measure`)
- severity, raw_value, suggested_value, resolved

## 18. 최종 포지셔닝

이 제품은 **악보 편집기를 대체하는 제품이 아니다.**  
이 제품은 **"깨진 MusicXML"을 "쓸 수 있는 MusicXML"로 만드는 정제 레이어**다.

- MuseScore / Dorico / Sibelius와 경쟁하지 않고, 그 앞단의 병목을 해결
- 코드 심볼 처리에서 현재 유일한 상용 솔루션 포지션
- 장기적으로는 기존 OMR 엔진 출력 위에 얹히는 후처리 레이어로 확장 가능
